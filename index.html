Qué es Loopbot

No es un producto para un dominio específico. Es el proceso por el cual se construye un agente operativo para cualquier dominio.

Loopbot es un framework para construir agentes LLM operativos. El agente arranca vacío — como un empleado nuevo — y aprende el negocio del humano a través de la interacción. Con el tiempo, gana autonomía y produce valor real.

5 principios fundamentales

01

El agente arranca vacío y aprende con la interacción

No hay configuración previa del dominio. El agente no sabe nada del negocio el día uno. Aprende haciendo preguntas, recibiendo correcciones, y observando patrones. El conocimiento se acumula como skills, reglas, y nodos en un grafo de objetivos. El patrón central es: detectar que no sé, registrar la respuesta, aplicar la próxima vez sin preguntar.

02

Autonomía creciente por skill, no global

La confianza se gana por capacidad específica, no como un switch general. El agente puede ser autónomo para confirmar visitas pero necesitar consulta para contraofertas. Cada skill tiene su propio nivel de autonomía que sube con aprobaciones y baja con correcciones. La autonomía no es un número que sube — es una consecuencia de que el skill tiene suficiente información para que el humano confíe en dejarlo actuar solo.

03

Configuración sobre código

El comportamiento se define con prompts, archivos de configuración, y reglas — no reescribiendo lógica. Cambiar cómo opera el agente es editar un archivo, no modificar TypeScript. La complejidad se mueve de código a configuración.

04

Loop universal, skills como diferenciador

El ciclo despertar, evaluar, actuar, persistir, dormir es siempre el mismo. Lo que cambia entre una inmobiliaria y una aseguradora son los skills, las entidades, y los criterios de priorización. El loop es la infraestructura; los skills son el negocio.

05

Proactividad vía detección de gaps

El agente no inventa trabajo. La proactividad nace del gap entre el estado actual y los objetivos estratégicos durante el heartbeat periódico. Si todo va según plan, el heartbeat es silencioso. Si hay un gap — un cliente sin respuesta hace 48 horas, una renovación próxima sin cotizar — el agente propone acciones para cerrarlo.

Qué produce

El resultado exitoso es un agente configurado y operativo para un negocio específico.

Identidad

Quién es el agente, qué negocio opera, y qué reglas sigue.

Capacidades

Skills de negocio que saben ejecutar tareas del dominio: coordinar visitas, cotizar seguros, hacer follow-ups.

Consciencia situacional

Un grafo de objetivos poblado — estratégicos, tácticos, operativos — que le dice al agente dónde están las cosas y qué falta hacer.

Guardrails

Reglas determinísticas que limitan lo que el agente puede hacer sin consultar al humano.

Autonomía calibrada

Cada skill con el nivel de autonomía que el humano aprobó a través del uso.

Cómo funciona

Meta-agente

El Regent

No opera el negocio — construye al agente que lo opera. Sabe hacer onboarding, crear skills, estructurar el grafo, incorporar feedback, y detectar qué le falta al agente. Es como un sistema operativo que sabe instalar aplicaciones.

Operador

El Agente

Recibe mensajes, consulta el grafo, ejecuta skills, y responde. Es el empleado que atiende clientes, hace follow-ups, y escala al humano cuando no sabe.

Ciclo de vida

Día 1

El regent hace la primera conversación. Pregunta qué hace el negocio, quiénes son los actores, qué problemas quiere resolver el humano. Todo es onboarding.

Primeras semanas

El regent identifica entidades, procesos, y patrones. Escribe la configuración del agente, crea los primeros skills, configura canales. 70% onboarding, 30% operación incipiente.

Operación

El onboarding se completa gradualmente — no hay switch. El regent pasa a modo calibración: incorpora feedback, escala autonomía de skills que funcionan bien, detecta gaps, propone skills nuevos. El agente opera el negocio.

Tres modos de operación

←

Reactivo

Un mensaje llega. El agente evalúa, ejecuta el skill correspondiente, responde.

↻

Heartbeat

Cada N minutos, el agente se despierta, consulta el grafo, detecta gaps, actúa o calla.

&clock;

Tarea programada

El agente programa una acción futura con contexto. Un scheduler la dispara a la hora indicada.

Feedback como motor de aprendizaje

El humano siempre tiene prioridad absoluta. Pero cada override es también feedback implícito:

→

Override puntual — Se registra en el grafo. Volátil, situacional.

→

Patrón recurrente — Se captura como criterio en el skill. Permanente, general.

→

Corrección de procedimiento — Se modifica el skill o se crea una regla nueva.

Dos planos: el grafo para lo puntual, el skill para lo que se generaliza.
